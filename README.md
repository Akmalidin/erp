## Возможности системы

### Каталог товаров
- Хранение товаров с полями: название, OEM номер, артикул, бренд, категория, цена закупки
- **Матрица авто** — привязка товаров к маркам и моделям автомобилей
- **Уровни наценок** — настраиваемые прайс-листы (Розничная, Опт, Дилер и т.д.)
- **История цен** — отслеживание изменений закупочных цен по времени
- **Умный поиск** — по OEM, артикулу, названию, бренду, марке/модели авто
- **Импорт прайс-листа** из Excel/CSV с маппингом колонок и редактированием перед загрузкой
- **Офлайн-очередь импорта** — при отсутствии интернета прайс сохраняется, загружается автоматически при подключении
- Печать ценников и прайс-листа (PDF)
- Фильтрация по категории, остатку, активности

### Склад
- Учёт остатков по складам и локациям
- История движений (приход, расход, перемещение, возврат)
- **Инвентаризация** — сравнение фактических и учётных остатков
- Перемещение товара между складами
- Автоматическое списание при продаже

### Продажи / Заказы
- Создание заказов с позициями, скидками (%), фиксированной суммой
- **POS Касса** — быстрый режим продажи (поиск товара, добавление, чек)
- Статусы заказов: Новый → В обработке → Завершён / Отменён / Возврат
- **Принять оплату** — прямо со страницы заказа, поддержка методов:
  - Наличные, Карта, Перевод, **QR mBank** (с загрузкой фото чека)
- История платежей по каждому заказу
- Долг по заказу — автоматически рассчитывается
- **Печать чека** (80мм термопринтер)
- **Печать накладной** (А4): полный состав, сумма прописью, имя менеджера, имя покупателя

### CRM / Клиенты
- Карточка клиента: контакты, компания, адрес, персональная скидка
- **Вкладки без перезагрузки**: Информация, Заказы, Платежи, Портал
- История заказов и платежей клиента
- Учёт долга клиента
- Принятие оплаты прямо из карточки клиента

### Клиентский портал
- Клиент входит по номеру телефона + пароль
- Просмотр своих заказов и их статусов
- Каталог товаров с ценами (розничный уровень)
- Корзина и оформление заказа
- Переписка с менеджером по позициям заказа (заметки)
- Уведомления менеджеру о действиях клиента
- Управление доступом (вкл/выкл, смена пароля) из карточки клиента

### Закупки
- Создание заказов поставщику
- Список поставщиков
- Импорт закупок из Excel

### Финансы
- **Кассовые смены** — открытие/закрытие смены, итоги
- **Расходы** — категории расходов, журнал трат

### Аналитика / Отчёты
- Сводный дашборд: выручка, заказы, долги, топ товаров
- Отчёт по продажам за период
- **ABC-анализ** товаров по выручке

### Панель суперадмина
- Управление всеми пользователями системы
- Блокировка / разблокировка аккаунтов
- Смена паролей
- Просмотр данных любого пользователя

### Интерфейс
- Тёмная и светлая тема (переключатель в навбаре)
- Сворачиваемый сайдбар (кнопка ☰ для всех мониторов)
- Адаптивный дизайн (мобильные + десктоп)
- Глобальный поиск
- Toast-уведомления

## Быстрый старт

### Требования
- Python 3.10+
- pip

### Установка

```bash
# Клонировать репозиторий
git clone https://github.com/Akmalidin/erp.git
cd erp

# Установить зависимости
pip install -r requirements.txt
```

## Десктоп-версия

Система запускается как **десктопное приложение** — без установки сервера, работает офлайн.

### Запуск без сборки

```bash
python run_server.py
```

Что происходит при запуске:
1. Проверяется интернет-соединение
2. Если **онлайн** → подключается к PostgreSQL на сервере
3. Если **офлайн** → использует локальную SQLite базу
4. Применяются миграции
5. Открывается Chrome/Edge в **app-режиме** (без адресной строки, как нативное приложение)
6. Фоновый поток мониторит подключение каждые 20 сек

### Сборка EXE

```bash
build.bat
```

Скрипт автоматически:
1. Устанавливает зависимости и PyInstaller
2. Собирает статические файлы
3. Компилирует `AutoPartsERP.exe` через PyInstaller
4. Создаёт `dist/Запустить AutoParts.bat`

Готовый файл: `dist/AutoPartsERP.exe`

При запуске EXE:
- Данные хранятся в `%LOCALAPPDATA%\AutoPartsERP\` (db.sqlite3, media)
- Браузерный профиль сохраняется между сессиями
- Не требует Python или каких-либо предустановленных компонентов

## Офлайн/Онлайн синхронизация

### Как это работает

```
Офлайн (нет интернета)          Онлайн (есть интернет)
──────────────────────           ─────────────────────────
SQLite (локальная БД)    ←──→   PostgreSQL (сервер)
  ↓                                      ↑
SyncLog (журнал                  Синхронизация при
изменений с метками              подключении
времени)
```

### Автоматическое уведомление

Когда появляется интернет — снизу экрана появляется уведомление:

> 🌐 **Появился интернет — переключиться на PostgreSQL?**  
> Офлайн-изменений: 42, импортов: 1  
> [ **Да, переключить** ] [ Позже ]

При нажатии **«Да»**:
1. Все локальные изменения (из `SyncLog`) загружаются на PostgreSQL
2. С PostgreSQL скачиваются изменения, произошедшие за время офлайна
3. **Конфликты** разрешаются по времени `updated_at` — побеждает более новая запись
4. Офлайн-импорты прайсов загружаются пакетом
5. Страница перезагружается с подключением к PostgreSQL

### Офлайн-импорт прайса

При загрузке Excel/CSV без интернета:
- Прайс сохраняется в очередь (`PendingImport`)
- Показывается сообщение: *«Сохранён в очередь — загрузится при подключении»*
- При синхронизации все ожидающие импорты загружаются **пакетом** (не по одному товару)

### Разрешение конфликтов

Если один и тот же товар/заказ изменён и в SQLite (офлайн), и в PostgreSQL (другой менеджер):
- Сравниваются поля `updated_at` обеих записей
- Запись с **более поздним временем** перезаписывает более раннюю
- Все операции логируются с точным временем UTC

## Деплой на VPS

### Переменные окружения (`.env`)

```env
DJANGO_SECRET_KEY=your-secret-key-here
DJANGO_DEBUG=False
DJANGO_ALLOWED_HOSTS=yourdomain.com,www.yourdomain.com,your-ip

# PostgreSQL
DB_NAME=erp_db
DB_USER=erp_user
DB_PASSWORD=your-db-password
DB_HOST=localhost
DB_PORT=5432
```

### Деплой (из Windows)

```bash
cd C:/Users/VERTEX/Desktop
python update.py
```

Скрипт: SFTP-загрузка изменённых файлов → `migrate` → рестарт gunicorn.

## Переменные окружения

| Переменная | По умолчанию | Описание |
|------------|-------------|----------|
| `DJANGO_SECRET_KEY` | `insecure-key` | Секретный ключ Django |
| `DJANGO_DEBUG` | `True` | Режим отладки |
| `DJANGO_ALLOWED_HOSTS` | `*` | Разрешённые хосты |
| `DB_NAME` | — | Имя PostgreSQL БД (если не задано → SQLite) |
| `DB_USER` | `erp_user` | Пользователь PostgreSQL |
| `DB_PASSWORD` | — | Пароль PostgreSQL |
| `DB_HOST` | `localhost` | Хост PostgreSQL |
| `DB_PORT` | `5432` | Порт PostgreSQL |
| `DB_MODE` | `sqlite` | Устанавливается автоматически (`sqlite` / `postgres`) |

> **Совет:** Для локальной разработки не задавай `DB_NAME` — система автоматически использует SQLite.
